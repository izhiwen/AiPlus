# Owner Gates

Stop for explicit Owner approval before:

- publishing, pushing, tagging, releasing, or deploying
- creating or modifying remote repositories
- global install or global config edits
- contacting external accounts
- telemetry, upload, remote service calls, MCP/App integrations
- using private project data outside the active target
- exposing private data externally or uploading it
- using private data not explicitly provided or named by the Owner for the task
- exposing secrets, tokens, or private paths
- claiming safety, compliance, legal, product-quality, or release-readiness guarantees
- treating simulated pressure-tests as real user research

Local project file edits are allowed only within the task scope requested by the Owner.
