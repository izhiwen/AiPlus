# v0.5.2 Known Gaps

## Overview

This document tracks known gaps, technical debt, and deferred work identified during the v2.1 hardening cycle. Items are categorized by priority and estimated effort.

## Critical Gaps (P0)

### 1. Capsule Recovery Edge Cases
- **Gap**: `compact resume` fails when capsule contains malformed UTF-8
- **Impact**: Recovery blocked for sessions with binary artifacts
- **Workaround**: Manual deletion of `.codex/compact/context-capsule.json`
- **Effort**: 2-3 days
- **Owner**: Compact Reminder team

### 2. Velocity Retention Policy
- **Gap**: No automatic purge of velocity records older than 90 days
- **Impact**: Unbounded disk growth in long-running projects
- **Workaround**: Manual `aiplus velocity purge --older-than 90d`
- **Effort**: 1-2 days
- **Owner**: Agent Velocity team

## High Priority Gaps (P1)

### 3. Cross-Module Conflict Detection
- **Gap**: Doctor scans run in isolation; conflicts between compact-reminder and agent-memory schemas not detected
- **Impact**: Silent schema incompatibilities after independent updates
- **Workaround**: Manual review of `docs/stabilization/v2.1-upgrade-path.md`
- **Effort**: 3-5 days
- **Owner**: Platform team

### 4. Redaction Coverage
- **Gap**: `raw transcript` pattern only catches English transcripts
- **Impact**: Non-English transcripts may leak into memory records
- **Workaround**: Locale-specific regex patterns in `redaction.rs`
- **Effort**: 2-3 days
- **Owner**: Agent Memory team

### 5. Consultant Team TOML Validation
- **Gap**: `consultant-team.default.toml` schema not enforced at runtime
- **Impact**: Invalid consultant configs silently ignored
- **Workaround**: Pre-commit validation via `aiplus doctor`
- **Effort**: 1-2 days
- **Owner**: Auto Team Consultant team

## Medium Priority Gaps (P2)

### 6. Test Coverage
- **Gap**: Parity tests missing for Windows paths and permissions
- **Impact**: Platform-specific regressions undetected
- **Workaround**: CI matrix expansion
- **Effort**: 3-4 days
- **Owner**: QA team

### 7. Documentation Localization
- **Gap**: v2.1 stabilization docs only available in English
- **Impact**: Chinese-speaking teams lack upgrade guidance
- **Workaround**: Machine translation + manual review
- **Effort**: 2-3 days
- **Owner**: Docs team

### 8. Performance Benchmarks
- **Gap**: No baseline for `memory_doctor()` on >10k records
- **Impact**: Performance regressions in large repositories
- **Workaround**: Manual profiling with `cargo bench`
- **Effort**: 2-3 days
- **Owner**: Agent Velocity team

## Low Priority Gaps (P3)

### 9. Schema Version Migration
- **Gap**: No automated migration from MEMORY_SCHEMA_VERSION_V1 to V2
- **Impact**: Legacy projects require manual migration
- **Workaround**: `aiplus memory migrate --from v1 --to v2`
- **Effort**: 1-2 days
- **Owner**: Agent Memory team

### 10. Telemetry Opt-out
- **Gap**: No granular telemetry controls (all-or-nothing)
- **Impact**: Privacy-conscious users cannot selectively disable metrics
- **Workaround**: Environment variable `AIPLUS_TELEMETRY=0`
- **Effort**: 1-2 days
- **Owner**: Platform team

## Deferred Work

The following items were scoped out of v2.1 and scheduled for v0.6.0:

- **Cloud sync for agent memory** (requires infrastructure design)
- **Vector database integration** (requires benchmarking)
- **Payment processing** (requires compliance review)
- **Voice interface** (requires speech-to-text provider selection)
