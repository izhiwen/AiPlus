# AiPlus v2.1 Upgrade Path: Installed Binary → Source Build

## Goal
Migrate from a stale release binary (`~/.local/bin/aiplus`) to a local source build so v2.1 features (compact resume capsules, memory doctor deep scans, profile bundle schema validation, decision ledger extraction) are available.

## Why Source Build Is Required
- v2.1 features are implemented in the working tree but not yet released.
- Running the installed binary will load the old codebase and silently skip new code paths.
- The authoritative binary for local development is `target/release/aiplus` built from this repo.

## Safety Boundaries
- No config migration is required.
- Profiles in `~/.config/aiplus/profiles/`, project-local `.aiplus/`, and `.codex/compact/` are preserved.
- No global agent config edits, no telemetry, no cloud sync, no secrets copied.

## Steps

### 1. Verify current binary

```bash
which aiplus
aiplus --version
```

If the path is `~/.local/bin/aiplus` and the version is older than the working tree, proceed.

### 2. Build from source

```bash
cd /Users/steve/Dropbox/Project/AiPlus/aiplus-public
cargo build -p aiplus-cli --release
```

### 3. Replace the installed binary (optional but recommended)

```bash
ln -sf $(pwd)/target/release/aiplus ~/.local/bin/aiplus
```

Or add an alias in your shell profile:

```bash
alias aiplus='/Users/steve/Dropbox/Project/AiPlus/aiplus-public/target/release/aiplus'
```

### 4. Verify the new binary

```bash
aiplus --version
aiplus compact resume --dry-run
aiplus memory doctor
aiplus profile doctor
```

You should see v2.1 output such as `context_capsule=loaded` or `decisions_loaded=N` in the compact resume path.

### 5. Keep the source tree up to date

Before each session, pull the latest changes and rebuild:

```bash
git pull
cargo build -p aiplus-cli --release
```

## Rollback

If you need to revert to the previous release binary:

```bash
# If you kept the old binary as a backup
mv ~/.local/bin/aiplus.backup ~/.local/bin/aiplus

# Or reinstall the last release from the original install method
```

## Troubleshooting

| Issue | Fix |
|---|---|
| `cargo: command not found` | Install Rust via rustup |
| Build fails with missing crates | Run `cargo fetch` then retry |
| `aiplus` still points to old binary | Check `which aiplus` and update `PATH` or alias |
| Profile/memory checks fail after upgrade | The data is safe; run `aiplus profile doctor` and `aiplus memory doctor` to see new validation rules |
