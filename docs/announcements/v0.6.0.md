# AiPlus v0.6.0 — Release Announcement

**Released**: 2026-05-17
**Tag**: [v0.6.0](https://github.com/izhiwen/AiPlus/releases/tag/v0.6.0)

## What's new

Three improvements end users feel immediately:

### 1. Plain-language role switching
Mid-session, say "you are CEO", "take the reviewer role", or "switch
to PI" and the agent re-binds to that role with its memory loaded. No
CLI command needed. Works in Codex, Claude Code, and OpenCode
interactive mode.

### 2. Smarter safety checks
Before doing anything risky (deleting files, publishing changes,
running protected commands), the agent now understands what you're
actually asking for, not just the words you typed. Rephrasing or
putting things in quotes can no longer trick it into bypassing the
check.

### 3. Team works ~2× faster
The review step and the QA step now run side by side instead of one
after the other. Each role's workspace stays ready between tasks
instead of being rebuilt every turn. Typical iterations: ~8-10 min
instead of ~15-20.

Plus for applied-economics researchers using AiEconLab:
- Two new specialist roles: **DoF Auditor** + **R&R Strategist**
- LLM-as-Measurement correlation heatmap in the main README
- Per-runtime role-switch notes in the adapter docs

## Install / upgrade

Fresh install:
```bash
curl -fsSL https://raw.githubusercontent.com/izhiwen/AiPlus/main/install.sh | bash
```

Update an existing install:
```bash
aiplus update
```

Then in your project:
```bash
aiplus install claude-code   # or codex / opencode / all
```

### Show HN

**Title**: Show HN: AiPlus v0.6.0 — Rust toolchain for pair-programming with AI agents

**Body**:
```
I've been pair-programming with AI coding agents (Codex, Claude Code,
OpenCode) full-time for the better part of a year. AiPlus is the Rust
toolchain I built to treat seven coordination failures I kept hitting:
cross-session memory loss, /compact timing, multi-agent collision,
AI-native time estimation (not human-hour anchored), plan-time blind
spots, single-agent role drift, and re-handing API keys every session.

v0.6.0 ships three improvements:

1. **Natural-language role switching**: mid-session, say "you are CEO"
   or "take the reviewer role" and the agent re-binds itself. Works in
   Codex, Claude Code, and OpenCode interactive mode. No CLI command.

2. **Semantic dispatch gate**: the coordinator parses intent
   (verb-object semantic analysis) before risky operations. Quoting
   or rephrasing can no longer smuggle a destructive command past it.

3. **~2× faster dispatch cycle**: reviewer and QA dispatch run as
   parallel sidecars; per-role worktrees pool with shared build cache.
   Typical iteration: 8-10 min instead of 15-20.

There's also an optional opt-in research-team module (AiEconLab) for
applied economists doing data work, replication packages, and referee
responses.

Honest meta-frame: I used AI agents to build the toolchain that
manages AI agents.

Repo: https://github.com/izhiwen/AiPlus
AiEconLab: https://github.com/izhiwen/AiEconLab

Bug reports and architecture critique welcome.
```
