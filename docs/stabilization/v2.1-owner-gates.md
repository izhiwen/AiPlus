# v2.1 Owner Gates

**Version:** v2.1  
**Status:** Planning

---

## Hard Gates (Always Require Owner Approval)

The following actions require explicit Owner approval for v2.1:

- `git push` to main
- `git tag` creation
- GitHub Release creation
- Artifact upload
- crates.io publish
- npm publish
- Homebrew publish
- Marketplace publish
- Global config edits
- Telemetry addition
- Cloud sync implementation
- Vector DB implementation
- Payment/voice features

## Soft Gates (Require Approval if Changing Behavior)

The following require Owner approval if they change existing behavior:

- Compact resume path changes (Task 1)
- Decision extraction scope expansion (Task 2)
- Memory record deletion features (Task 3)
- Profile schema validation rules (Task 4)
- README upgrade instructions (Task 5)

## Automatic Approvals (No Gate)

The following do NOT require Owner approval:

- Local development and testing
- Documentation-only changes (if synthetic/redacted)
- Unit test additions
- Code formatting and lint fixes
- Bug fixes with existing test coverage
- v2.1 planning docs (this file, task cards, etc.)

## Decision Matrix

| Action | Owner Approval | Notes |
|--------|---------------|-------|
| Implement Task 1 code | No | But PR requires approval |
| Merge Task 1 to main | Yes | Owner reviews PR |
| Create v2.1 tag | Yes | Explicit approval |
| Release v2.1 artifact | Yes | Explicit approval |
| Update docs only | No | If no private content |
| Add unit tests | No | |
| Refactor internals | No | If no behavior change |

## Escalation Path

If a developer is unsure whether an action requires Owner approval:

1. Default to "requires approval"
2. Document the proposed change
3. Ask in the next sync or via task card comment
4. Do not proceed until clarified
